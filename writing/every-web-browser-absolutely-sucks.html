every web browser absolutely sucks

2026-04-18 · placeholder content (original by luke smith)

The title explains it all—you don't even have to read further.

There are no good, even passable web browsers. None. Not a single one even comes close.

The curious thing is this: making a good browser should be easy! Among existing web browsers, you could assemble all the necessary parts for a passable browser. Instead, people assembled 90% good stuff and 10% junk.

This post lists:

  1. Features a passable browser must have
  2. Features a good browser must have

No browser currently has all traits of even a passable browser, but here they are for the record.

features a passable browser must have

it must actually work on the modern web

Terminal browsers like lynx and w3m don't qualify. While useful for scripting and email, they exclude most modern web content. Niche browsers like Dillo also fail—they cannot handle contemporary CSS properly.

free and open source software

There's no justification for using a closed-source internet browser. The source code must be publicly auditable.

no unsolicited connections

This should be the easiest requirement to meet, yet it's the rarest feature:

  • Don't automatically connect to Google, Brave, analytics pages, or Cloudflare on startup or during browsing
  • Don't automatically connect to update servers
  • Don't send analytics or create analytics databases
  • Don't prompt for bug report opt-ins

If analytics are desired, relegate requests to obscure Settings pages. However, users typically won't consent—they don't allow monitoring from file managers, email clients, or music players, and browsers handling sensitive information deserve even stricter standards.

ad-blocking must come with the browser

No one browses to view advertisements. A sensible browser should block ads universally, just as a rented server should come with an operating system and firewall.

Ad-blocking should be universal. HTTPS Everywhere should also be a default browser feature with no use cases for HTTP preference significant enough to leave enabled by default.

Brave blocks ads by default and includes HTTPS Everywhere. Qutebrowser has a basic hosts-based ad-blocker but provides an ad-filled experience. GNU Icecat includes sensible add-ons by default.

basic options

Why haven't Chrome and its clones added an option to disable history storage in a decade?

  • Not Incognito mode (which keeps cookies but stores history)
  • Not just hiding previous suggestions while storing everything
  • An actual option to prevent history from appearing when typing URLs

Firefox and Brave offer this. Ungoogled Chromium doesn't. The mandatory history feature seems like subtle social engineering: "you shouldn't have that choice; everything should be stored for reference."

internet browser history is really obsolete

Modern web browsing makes history largely unnecessary because:

  1. Users visit only Facebook or two other sites
  2. Bookmarks exist and are widely used
  3. Search engines can locate sites
  4. Many users delete history anyway

no clutter in the browser experience and neutrality

Brave blocks ads by default—praised initially—but continually adds niche features like cryptocurrency gadgets that make unsolicited connections. While disableable, these aren't browsing-related.

Most browsers feature distracting start pages advertising history or suggesting sites. A fresh window should display a clean page unless users have set a custom homepage.

Browsers should be neutral, not personalized. They shouldn't advertise new features or change based on visited sites.

do not clutter home

Browsers create unnecessary directories in home folders. Standard directories exist:

  • ~/.config/ — configuration files
  • ~/.cache/ — cache
  • ~/.local/share/ — share files

These prevent home directory clutter. Browser developers ignore this.

Chromium creates ~/.pki/. Firefox-based browsers force ~/.mozilla/ and others. Librewolf adds ~/.librewolf/.

Pale Moon deserves special mention: ~/.Moonchild Productions with capital letters and whitespace—disrespectful to Unix systems.

must be written in a sensible language

Qutebrowser fails here. While Python allows easy configuration customization, it's slow, buggy, and resource-intensive compared to C or C++ browsers. Compilation in Cython or Go might help, but current performance doesn't meet mainstream needs.

Qutebrowser handles most aspects well; with more CPU and RAM, it might be usable.

features a good browser must have

Beyond minimum requirements:

a config file

Only 2% of people understand configuration files, but every browser needs one—alongside Settings menus for non-technical users.

Once configured, users can move one file to replicate settings on new computers. System administrators could distribute sensible defaults through a single text file.

Configuration files should handle:

  • Add-on lists with source links (like vim plugin managers)
  • Key bindings and custom shortcuts
  • Bookmarks
  • Search engine prefixes
  • Graphical settings, colors, tab organization
  • Font and language

Config files enable sharable, editable settings and solve tech support issues.

actually…

After implementing configuration files, other features follow naturally. For example, vim-like key-bindings for mouseless browsing work through custom key-binds with link hints.

With sensible defaults and a configuration file, browsers gain the same usability, customizability, and non-obtrusiveness as other computer programs.

summary

A browser must:

  1. Actually work
  2. Be free and open source software
  3. Make no unsolicited connections
  4. Block unsolicited ads with sensible defaults
  5. Offer sensible history and cookie options
  6. Avoid irrelevant browsing features
  7. Respect filesystem standards
  8. Use fast, resource-efficient languages
  9. Include a configuration file

Tell me when a browser meets these requirements.
